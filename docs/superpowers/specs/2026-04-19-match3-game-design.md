# 消消乐游戏设计规格书

## 1. 项目概述

- **项目名称**: 萌宠消消乐 (Cute Pet Match-3)
- **技术栈**: React + Framer Motion
- **游戏类型**: 三消益智游戏
- **核心玩家**: 休闲游戏爱好者

## 2. 游戏规格

### 棋盘与元素

| 参数 | 值 |
|------|---|
| 棋盘大小 | 10 × 10 格子 |
| 元素类型 | 5 种动物图标 |
| 匹配规则 | 3 个或以上相同元素连成一线（横向或纵向） |

### 5种动物图标

| ID | 名称 | 主色调 | 图标特征 |
|----|------|--------|----------|
| cat | 橘猫 | #FF8C42 | 圆脸、眯眼微笑、橘色毛 |
| dog | 柯基犬 | #C4A484 | 大耳朵、蜜桃臀轮廓 |
| rabbit | 小白兔 | #FFB6C1 | 长耳朵、粉色鼻子 |
| bear | 小熊 | #8B5A2B | 圆耳朵、蜂蜜表情 |
| panda | 熊猫 | #2C2C2C | 黑眼圈、竹子表情 |

## 3. 游戏规则

### 3.1 经典三消

- 玩家可以交换两个相邻（上下左右）的元素
- 交换后必须形成至少一个匹配（3个或以上相同元素连成一线）
- 如果交换后无法形成匹配，元素自动返回原位

### 3.2 连锁反应

- 匹配消除后，上方元素下落填补空位
- 新的匹配形成后会继续消除
- 连锁反应可持续直到没有新的匹配生成为止
- 连锁反应期间不消耗步数

### 3.3 步数限制

- 每关开始时有固定步数
- 每次有效交换（形成匹配）消耗 1 步
- 步数归零时游戏结束

### 3.4 目标通关

- 每关设置 2-3 个组合目标
- 目标类型：消除指定数量的某种动物
- 所有目标达成后过关
- 未达成目标且步数归零则游戏失败

## 4. 关卡系统

### 关卡配置结构

```typescript
interface Target {
  type: ElementType;  // 动物类型
  count: number;      // 需消除数量
}

interface LevelConfig {
  id: number;          // 关卡编号
  targets: Target[];   // 目标列表（2-3个）
  moves: number;       // 初始步数
}
```

### 关卡示例

| 关卡 | 目标 | 步数 |
|------|------|------|
| 1 | 消除 10 只橘猫 | 20 |
| 5 | 消除 15 只橘猫 + 消除 10 只柯基犬 | 25 |
| 10 | 消除 20 只小白兔 + 消除 15 只小熊 | 30 |

### 关卡表

- 总共 50 关
- 难度递增：目标数量增加、步数相对减少
- 后期关卡需要策略性规划才能通关

## 6. 游戏状态设计

```typescript
type ElementType = 'cat' | 'dog' | 'rabbit' | 'bear' | 'panda';
type GameStatus = 'idle' | 'playing' | 'won' | 'lost';

interface GameState {
  board: (ElementType | null)[][];  // 10x10 棋盘，null 表示空位
  level: number;                     // 当前关卡编号
  moves: number;                     // 剩余步数
  score: number;                     // 累计分数
  targets: Target[];                 // 当前关卡目标
  progress: Record<ElementType, number>; // 各元素已消除数量
  gameStatus: GameStatus;           // 游戏状态
}
```

## 7. 核心算法流程

### 交换与匹配流程

```
1. 玩家选择并交换两个相邻元素
2. 检查交换后是否形成匹配
   - 无匹配：元素自动返回，步数不变
   - 有匹配：进入消除流程
3. 消除流程：
   a. 播放消除动画
   b. 更新分数
   c. 更新目标进度
   d. 上方元素下落填补空位
   e. 检查是否有新的匹配
      - 有：返回步骤 3a（连锁反应）
      - 无：步数 -1，进入步骤 4
4. 检查游戏状态
   - 目标全部达成 → 游戏胜利
   - 步数归零且目标未达成 → 游戏失败
   - 否则继续等待玩家操作
```

## 8. 动画设计

| 动画名称 | 触发时机 | 持续时间 | 效果 |
|----------|----------|----------|------|
| swap | 交换两个元素 | 0.2s | 位置平滑过渡 |
| pop | 元素被消除 | 0.3s | 缩放放大 + 渐变消失 |
| fall | 元素下落 | 0.4s | 弹性下落动画 |
| scorePopup | 得分时 | 0.5s | 数字飘字 + 放大 |

## 9. 计分规则

| 消除元素数 | 单次得分 |
|------------|----------|
| 3 个 | 30 分 |
| 4 个 | 50 分 |
| 5 个 | 80 分 |
| 连锁加成 | 每增加一次连锁 +20 分 |
