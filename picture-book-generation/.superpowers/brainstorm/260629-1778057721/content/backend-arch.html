后端架构确认

关于数据存储和用户系统

A

纯前端 + API 调用

前端直调 MiniMax API，无需用户登录，所有数据在前端内存中处理

B

Node.js 后端 + 本地存储

后端作为中间层转发 API 请求，绘本数据存储在服务器本地文件中

C

Node.js 后端 + 数据库

使用 SQLite/MySQL/PostgreSQL 等数据库持久化存储绘本和用户数据


选择 A 的理由

  • 架构简单，开发速度快
  • 无需管理服务器和数据库
  • 适合单人使用或临时展示

选择 B/C 的理由

  • 保护 API Key 不暴露在前端
  • 可以添加用户登录系统
  • 绘本数据可以保存和复用
  • 支持后续功能扩展
