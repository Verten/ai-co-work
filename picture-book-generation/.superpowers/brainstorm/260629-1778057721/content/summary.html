设计方案总结

请确认以下设计决策

1. 首页布局

卡片式选择 - 两个大卡片并排："对话生成" 与 "随机生成"

2. 对话生成模式

引导式多轮对话 - 系统逐步引导用户完善故事细节（主角→场景→情节→主题）

3. 随机生成要素（全部让用户选择）

  • 主角类型（小女孩/小男孩/小动物/怪物/机器人...）
  • 故事场景（森林/海底/太空/城市/农场/学校...）
  • 故事主题（冒险/友情/成长/幽默/勇气/亲情...）
  • 画风风格（卡通/水彩/油画/素描/动漫...）

4. 绘本预览与输出

翻页预览 + 打印 - 类似真实绘本的左右翻页效果，支持打印成册

5. 后端架构

Node.js 后端 + 数据库 - API Key 安全，绘本数据持久化存储

6. 用户系统

需要登录 - 用户注册/登录后，可以保存历史绘本、管理作品集

技术栈

  • 前端: React
  • 后端: Node.js
  • 数据库: SQLite/MySQL/PostgreSQL
  • AI: MiniMax API (文本生成 + 文生图)

绘本规格

  • 每本 10 页左右
  • 每页: 图片 + 文字
  • 画风: 漫画/元气/中世纪/水彩
  • 导出: 支持打印、下载 PDF
