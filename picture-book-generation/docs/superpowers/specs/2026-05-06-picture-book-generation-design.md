# 儿童绘本生成器 - 设计文档

## 1. 项目概述

**项目名称**: 童话绘本生成器
**项目类型**: Web 应用（前后端分离）
**核心功能**: 生成文字和关联图片的儿童绘本
**目标用户**: 家长、教师、以及需要为儿童创作绘本的用户

## 2. 功能模块

### 2.1 首页 - 模式选择

用户进入首页后，通过**卡片式选择**两种生成方式：

| 模式 | 描述 |
|------|------|
| 对话生成 | 用户描述故事大纲，系统引导式丰富成完整绘本 |
| 随机生成 | 用户选择绘本要素（主角、场景、主题、画风），系统生成绘本 |

### 2.2 对话生成模式

采用**引导式多轮对话**流程：

1. **主角设定** - 系统引导选择主角类型
   - 小女孩、小男孩、小动物（小熊/小兔/小狗等）、怪物、机器人等

2. **场景设定** - 选择故事发生场景
   - 森林、海底、太空、城市、农场、学校等

3. **情节发展** - 描述故事主要情节/问题

4. **主题选择** - 故事传达的主题
   - 冒险、友情、成长、幽默、勇气、亲情等

系统根据用户逐步输入的内容，调用大语言模型生成 10 页左右的完整绘本故事。

### 2.3 随机生成模式

用户一次性选择所有绘本要素：

| 要素 | 选项 |
|------|------|
| 主角类型 | 小女孩、小男孩、小熊、小兔、小狗、小猫、机器人、怪物等 |
| 故事场景 | 森林、海底、太空、城市、农场、学校、城堡、丛林等 |
| 故事主题 | 冒险、友情、成长、幽默、勇气、亲情、智慧、感恩等 |
| 画风风格 | 卡通、水彩、油画、素描、动漫 |

系统随机组合要素，调用大语言模型生成 10 页绘本。

### 2.4 绘本预览与输出

采用**翻页预览 + 打印**模式：

- 类似真实绘本的左右翻页效果
- 每页显示：AI 生成的图片 + 对应文字
- 支持操作：
  - 打印成册
  - 下载 PDF
  - 重新生成（基于当前故事重新生成图片）
  - 返回修改故事

### 2.5 用户系统

**需要用户登录**，支持：

- 用户注册（邮箱/用户名 + 密码）
- 用户登录
- 保存历史绘本列表
- 查看/删除历史绘本
- 作品管理（重命名、复制等）

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React (Vite) |
| 后端 | Node.js + Express |
| 数据库 | SQLite（轻量级，无需额外部署） |
| AI API | MiniMax API（文本生成 + 文生图） |

### 3.2 数据库设计

**用户表 (users)**

| 字段 | 类型 | 描述 |
|------|------|------|
| id | INTEGER | 主键 |
| username | TEXT | 用户名 |
| email | TEXT | 邮箱 |
| password_hash | TEXT | 密码哈希 |
| created_at | DATETIME | 创建时间 |

**绘本表 (storybooks)**

| 字段 | 类型 | 描述 |
|------|------|------|
| id | INTEGER | 主键 |
| user_id | INTEGER | 外键，所属用户 |
| title | TEXT | 绘本标题 |
| story_config | TEXT | 生成配置（JSON） |
| pages | TEXT | 页面内容（JSON数组） |
| status | TEXT | 状态：generating/completed/failed |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**页面结构 (pages JSON)**

```json
[
  {
    "page_number": 1,
    "text": "小熊噜噜第一次去上学...",
    "image_url": "https://..."
  },
  ...
]
```

### 3.3 API 设计

| 端点 | 方法 | 描述 |
|------|------|------|
| /api/auth/register | POST | 用户注册 |
| /api/auth/login | POST | 用户登录 |
| /api/auth/logout | POST | 用户登出 |
| /api/storybooks | GET | 获取用户绘本列表 |
| /api/storybooks | POST | 创建新绘本（触发生成） |
| /api/storybooks/:id | GET | 获取绘本详情 |
| /api/storybooks/:id | DELETE | 删除绘本 |
| /api/generate/chat | POST | 对话生成故事（流式） |
| /api/generate/random | POST | 随机生成故事 |

### 3.4 AI 调用流程

**文本生成** (`POST https://api.minimaxi.com/anthropic/v1/messages`)

- 输入: 用户描述/选择的要素
- 输出: 结构化的故事内容（10 页左右的文字）
- 模型: MiniMax-M2.7

**文生图** (`POST https://api.minimaxi.com/v1/image_generation`)

- 输入: 每页的故事文字描述 + 画风
- 输出: 对应的绘本图片
- 模型: image-01
- 画风: 漫画、元气、中世纪、水彩
- 尺寸: 3:4（竖版绘本比例）

## 4. 前端页面结构

```
/                     - 首页（模式选择）
/chat                 - 对话生成页面
/random               - 随机生成页面
/preview/:id          - 绘本预览页面
/profile              - 用户个人中心
/login                - 登录页

```


## 6. 绘本生成规格

| 项目 | 规格 |
|------|------|
| 每本页数 | 约 10 页 |
| 每页内容 | 1 张图片 + 对应文字 |
| 图片比例 | 3:4（竖版绘本） |
| 支持画风 | 漫画、元气、中世纪、水彩 |
| 导出格式 | PDF、打印 |
